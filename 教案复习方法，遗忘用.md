# 📂 教资科目三物理 · 终极复习 SOP (V4.0 标准作业程序)

**🏷️ Tags:** #教资备考 #高中物理 #教学设计 #SOP
**🎯 核心心法：** 纵向推进 (Top-down Design) | 主动踩坑 (Test-Driven) | 双轨输出 (Double-Track)

---

## 📌 核心准则 (The Golden Rule)
**绝对禁止“点代替面”！** 每一个 T0 级物理模块，必须经历从“三维目标 -> 重难点 -> 导入 -> 探究 -> 总结”的完整类（Class）构建。没有亲身踩过的坑，就无法形成肌肉记忆。

---

## ⚙️ 标准执行流 (Execution Flow)

### 🟢 阶段一：环境初始化 (Initialization)
* **触发条件**：教官下发新课名称及教材截图。
* **用户动作**：不看任何参考答案，纯靠物理直觉，用大白话敲出这节课的**【教学重点】**和**【教学难点（核心 Bug）】**。
* **系统响应**：教官校验重难点是否精准，确立本节课的主攻方向。

### 🟡 阶段二：全链路沙盘推演与主动踩坑 (Step-by-Step Debugging) —— 【核心环节】
* **执行逻辑**：教官按时间轴进行“连环逼问”，绝不提前剧透。用户逐一进行接口开发，主动试错。
    * **接口 1【新课导入】**：如何制造认知冲突？如何打破生活直觉？
    * **接口 2【探究设计】**：硬件怎么搭？变量怎么控？会有什么极其容易翻车的细节？（教官负责 Throw Exception 报错，用户负责打补丁）。
    * **接口 3【数据处理】**：如何化曲为直？如何运用奥卡姆剃刀等科学哲学思想引导学生？
* **输出成果**：在不断的拉扯和 Debug 中，生成最真实的《Bug 与踩坑日志》。

### 🔴 阶段三：双轨制最终编译 (Double-Track Build)
* **触发条件**：所有逻辑闭环跑通，Bug 全部修复。
* **系统动作**：教官负责将用户的碎片化输出，打包封装为符合考场格式的终极 Markdown 笔记。
* **交付物结构**：
    1. **The Dev Log (开发日志)**：记录踩坑点与修复思路。
    2. **Class A (Q12 微格片段库)**：针对特定痛点的 12 分师生对话剧本（约 200 字）。
    3. **Class B (Q13 重装机甲)**：包含三维目标、重难点、教学过程，且**必须带有【设计意图】**的 28 分全局教案（400-500 字）。

> 💡 **架构师批注**：教案不是背出来的，是基于物理第一性原理“生成”出来的。相信逻辑，相信直觉！
